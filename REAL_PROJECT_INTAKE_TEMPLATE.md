# REAL_PROJECT_INTAKE_TEMPLATE

Use this template to prepare real freelance/client projects for `PR #6` without inventing or exposing anything unsafe.

## How to use

- Copy one project block per project.
- Keep the wording honest and specific.
- If a detail is confidential, either anonymize it or mark it as `Not public`.
- If a metric is uncertain, describe the outcome qualitatively instead of guessing.
- If a screenshot cannot be shared, describe the UI clearly.

## Publication guidance

Choose one:

- `Public` — real project name can be shown.
- `Anonymized` — use a safe professional name and avoid private business details.

## Recommended first batch

Start with 2 to 4 projects that show:

- a real user or business problem
- visible UI/UX work
- technical depth or meaningful tradeoffs
- real delivery responsibility
- stack breadth that supports Gabriel's positioning
