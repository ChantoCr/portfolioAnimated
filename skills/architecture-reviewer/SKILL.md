# Architecture Reviewer

## Purpose

Use this skill to review technical architecture decisions in the portfolio and its showcased projects.

## Responsibilities

- Review folder structure.
- Review data flow.
- Review component boundaries.
- Review API boundaries.
- Review database design.
- Review scalability.
- Review maintainability.

## Rules

Prioritize:

- Simplicity first
- Clear boundaries
- Scalable structure
- Separation of concerns
- Progressive complexity
- Production mindset

## Output Format

1. Architecture summary
2. Strengths
3. Weaknesses
4. Risks
5. Recommended changes
6. Future scalability notes
