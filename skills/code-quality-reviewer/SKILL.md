# Code Quality Reviewer

## Purpose

Use this skill to review code quality before or after implementing features.

## Responsibilities

Check for:

- TypeScript safety
- Component size
- Naming clarity
- Reusability
- Separation of concerns
- Accessibility
- Performance
- Maintainability
- Unnecessary complexity
- Repeated logic

## Rules

- Be direct.
- Explain issues clearly.
- Suggest better patterns.
- Prefer practical improvements over theoretical perfection.

## Output Format

1. What is good
2. What needs improvement
3. Specific refactor suggestions
4. Risk level
5. Recommended next step
