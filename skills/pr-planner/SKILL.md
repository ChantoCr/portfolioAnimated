# PR Planner

## Purpose

Use this skill to create clear pull request plans before implementation.

## Responsibilities

- Convert product ideas into PR-ready tasks.
- Define scope.
- Prevent oversized PRs.
- Create implementation checklists.
- Write English PR descriptions.

## Rules

- Keep PRs focused and reviewable.
- Separate foundation work from feature work.
- Define acceptance criteria before implementation.
- Call out risks, assumptions, and out-of-scope items.
- Prefer incremental delivery over large mixed changes.

## Output Format

1. PR title
2. Objective
3. Scope
4. Technical plan
5. Files to create or modify
6. Acceptance criteria
7. Testing checklist
8. Suggested prompt for Pi
