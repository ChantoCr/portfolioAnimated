# AGENTS.md

## Project Name

AI-Powered Developer Portfolio OS

## Project Vision

An AI-powered portfolio operating system that showcases my projects, technical thinking, architecture decisions, UI/UX process, and software engineering growth through an interactive agent-driven experience powered by OpenAI, Skills, AGENTS.md, and Pi.

This is not a simple portfolio website. It is a full product experience designed to demonstrate senior-level thinking, product architecture, UI/UX skill, AI-assisted workflows, and real-world software engineering discipline.

## Primary Goal

Build a professional, cinematic, AI-powered portfolio platform for Gabriel Soto that impresses senior developers, recruiters, clients, and technical leaders.

The platform must show:

- Projects
- Case studies
- Technical architecture
- UI/UX process
- Coding philosophy
- Market positioning
- Skills and technologies
- AI-assisted development workflow
- Contact and professional information
- Interactive AI Agent Console

## Development Philosophy

Always build as if this were a production-grade product.

Prioritize:

- Clean architecture
- Type safety
- Reusable components
- Scalable folder structure
- Clear naming
- Strong UI hierarchy
- Accessibility
- Performance
- SEO
- Smooth animations
- Real-world maintainability

## Coding Rules

- Use TypeScript everywhere.
- Avoid `any` unless absolutely necessary.
- Use reusable components.
- Keep components small and focused.
- Separate UI, data, logic, and configuration.
- Prefer server components when possible.
- Use client components only when interactivity is required.
- Do not hardcode repeated content directly inside components.
- Use structured content files for projects, skills, timeline, and case studies.
- Every major feature should be easy to extend later.

## UI/UX Rules

The design must feel premium, cinematic, modern, and professional.

Use:

- Dark elegant interface
- Strong contrast
- Smooth transitions
- Large typography
- Clean spacing
- Subtle gradients
- Professional animations
- Interactive cards
- Scroll-based storytelling
- Minimal but powerful visual effects

Avoid:

- Generic portfolio templates
- Overloaded sections
- Random animations without purpose
- Weak visual hierarchy
- Too many colors
- Cheap-looking effects

## Animation Rules

Animations must feel intentional.

Use animations for:

- Hero entrance
- Section reveal
- Project cards
- Skill transitions
- AI console interactions
- Architecture diagrams
- Timeline progression
- Hover states
- Page transitions

Avoid animations that hurt readability or performance.

## AI Agent Rules

The AI agent must never hallucinate about Gabriel.

The agent should answer only from:

- Project data
- CV content
- Case studies
- Skill files
- Architecture notes
- Approved professional summary

If information is missing, the agent should say that the information has not been provided yet.

## Feature Development Process

For every feature:

1. Understand the product goal.
2. Identify the user value.
3. Propose the UI/UX approach.
4. Define required components.
5. Define required data structure.
6. Implement cleanly.
7. Add responsive behavior.
8. Add animation only where useful.
9. Check accessibility.
10. Suggest improvements.

## Branch / PR Rules

Every pull request should include:

- Summary
- What changed
- Why it changed
- Screenshots if UI changed
- Testing notes
- Future improvements
